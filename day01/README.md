## 데이터베이스 정규화

#### 개념

데이터베이스 정규화는 데이터베이스의 설계를 재구성하는 테크닉이다. 정규화를 통해 불필요한 데이터를 없앨 수 있고, 삽입/ 갱신/ 삭제 시 발생할 수 있는 각종 이상현상등을 방지할 수 있다.

#### 목적

1. 불필요한 데이터(data redundancy) 제거
2. 데이터를 논리적으로(logical) 저장

#### 제1 정규형(1NF)

- 데이터의 원자성

  - 모든 컬럼은 원자값(Atomic Value)을 갖는다.

    | NAME | REGION         |
    | ---- | -------------- |
    | Kim  | Seoul, Deajeon |

    위와 같이 Region 컬럼에 두개의 값이 들어가는 경우 1NF 위반이다.

#### 제2 정규형(2NF)

- 완전 함수적 종속

  - 키가 아닌 컬럼들은 기본키에 완전히 함수적 종속된다.

  - 기본키의 일부가 특정 컬럼의 값을 결정(부분적 종속)지을 수 없다.

    ex) 기본기카 (A, B)라고 했을 때 특정 컬럼 C가 A에 함수적으로 종속되어 있다면 제 2 정규화가 이루어 지지 않은 것이다.

완전 함수적 종속이 중요한 이유는 아래와 같은 문제점이 발생할 수 있기 때문이다.

![](http://databaser.net/moniwiki/pds/_ec_a0_95_ea_b7_9c_ed_99_94_ec_9d_98_ec_9d_b4_ed_95_b4/normalization11.jpg)

#### 제 3 정규형(3NF)

- 이행적 함수 종속 제거

  - 모든 컬럼들은 키가 아닌 컬럼에게 함수적으로 종속 될 수 없다.

    ex) 엔티티의 기본기카 (A)이고 키가아닌 컬럼 B 와 C가 있다고하자. B, C의 관계가 B -> C 일 때, A -> B -> C 가 성립하므로 이행적 함수 종속이 발생한다. 이런 경우 제 3 정규화가 이루어지지 않은 것이다. 

위의 ex을 볼때 A값이 없으면 B, C 데이터를 추가 할 수 없고(삽입이상), 데이터가 삭제될 경우 B, C의 관계가 같이 삭제 되고(삭제이상), B 또는 C의 값이 바뀌었을 때 해당하는 모든 데이터의 값을 갱신(갱신이상)해야하게 된다.
